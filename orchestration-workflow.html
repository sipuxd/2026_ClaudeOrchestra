01 Pre-Work — Security Scan
ENGINE SENDS SCAN REQUEST
Sends task description and project path to Security-1
↓ SDK query()
SECURITY-1 — PRE-SCAN
Scans entire target workspace before any work begins
Prompt injection patterns in files Hardcoded credentials, API keys, secrets Dependency integrity validation Sensitive area mapping (auth, db, env, crypto) Task complexity classification (SIMPLE/STANDARD/COMPLEX)
↓ clearance report + classification
ENGINE PARSES CLASSIFICATION
Security can reclassify: if SIMPLE, downgrades to simple pipeline. If COMPLEX, enables strict review.
02 Work — Implementation + Verification
ENGINE SENDS TASK TO WORKER-1
Task + approved requirements + security clearance boundaries
↓ SDK query()
WORKER-1 — IMPLEMENTS
Executes assigned work within cleared scope. Writes code, creates files, runs commands.
↓ implementation result
Requirements Verification Loop (max 2 passes)
WORKER-2 — VERIFIES
Checks Worker-1's output against task requirements. Read-only — never modifies code.
Outputs checklist: [x] met, [ ] unmet Verdict: COMPLETE or GAPS_FOUND
↓ verdict
COMPLETE
All requirements met — proceed to sweep
GAPS_FOUND
Worker-1 fixes gaps, Worker-2 re-verifies
↩ GAPS_FOUND: Worker-1 fixes unmet requirements, Worker-2 re-checks (max 2 passes)
git Auto-commit: "WIP: work phase complete"
03 Handoff — Security Sweep
ENGINE SENDS SWEEP REQUEST
Sends task + Worker-1 and Worker-2 summaries to Security-1
↓ SDK query()
SECURITY-1 — POST-WORK SWEEP
Validates all completed output before it reaches the Reviewer
Prompt injection in new/modified files Accidentally committed secrets or credentials Unauthorized dependencies added Scope adherence — only touches cleared areas
↓ verdict
APPROVED / FLAGGED
Proceed to Review
BLOCKED
Retry Work phase with updated constraints
↩ BLOCKED: Engine transitions Handoff → Work (increments revision counter, max 3)
git Auto-commit: "WIP: security sweep passed" (on APPROVED/FLAGGED)
04 Review — Quality Evaluation
ENGINE SENDS REVIEW REQUEST
Sends task + Worker summaries to Reviewer-1. Strict criteria for COMPLEX tasks.
↓ SDK query()
REVIEWER-1 — EVALUATES
Assesses quality only — does NOT evaluate security
Correctness — does it solve the task Code quality — structure, readability, patterns Completeness — gaps, edge cases, tests Integration — fits with broader codebase
↓ verdict
APPROVED
Pipeline complete — sessions kept alive for Q&A
REVISION_NEEDED
Retry Work phase with feedback
REJECTED
Restart from Pre-Work (full re-plan)
↩ REVISION_NEEDED: Engine transitions Review → Work (increments revision counter, max 3)
↩↩ REJECTED: Engine transitions Review → PreWork (increments rejection counter, max 2)
git Final auto-commit: task description (on APPROVED)
PIPELINE COMPLETE
All agent sessions remain alive for Q&A. Human can Push & Merge from the dashboard.
LOOP LIMITS (enforced by state machine)
Max 3 revisions (Handoff→Work or Review→Work) Max 2 rejections (Review→PreWork) Max 5 total backward transitions Max 2 verification passes per Work phase entry Exceeding any limit → errored state → human notified
